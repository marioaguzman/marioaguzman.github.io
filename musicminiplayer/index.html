Music MiniPlayer for macOS

A replica of the classic iTunes MiniPlayer
to control Apple Music on modern macOS.

macOS 26 Tahoe Support

With the release of macOS Tahoe, Music v1.6.0 no longer supplies the data required for Music MiniPlayer to operate correctly. Unfortunately, there is no technical workaround for this change. As a result, macOS Sequoia remains the last version fully supported by Music MiniPlayer.

Basic Controls


Music MiniPlayer has all the basic controls to control Apple Music and just enough information and status indicators to keep it glanceable. It’s there when you need it without need any interaction and you can even keep it floating above all your other windows.

Playlists


Unlike the original MiniPlayer, Music MiniPlayer allows you to play another playlist without having to open the main Apple Music window. Just slide open the drawer to pick a different playlist or control Shuffle and Repeat modes.

Live Streaming UI


When you’re listening to live audio such as Apple Music Hits or LA’s KIIS FM, Music MiniPlayer adapts its user interface to something suitable for streaming audio.

Craftsmanship


Music MiniPlayer is a 100% native, written in both AppKit and Swift. Nearly the entire UI is drawn in code using Core Graphics & Core Animation0 in order to achieve crisp details for both Retina and non-Retina displays.

Scrolling Marquee


The “Now Playing” display performs beautiful, synchronized marquee animations to display the current track or radio station.

Playlist Drawer


Switch open the drawer to reveal your Apple Music playlists or set your Shuffle or Repeat modes.

🛠 Technical stuff for the nerds

Music MiniPlayer uses Scripting Bridge and Distributed Notification Center to send and receive commands with Apple Music on macOS. 99%2 of the UI is written in Core Graphics and Core Animation to achieve crisp UI that is also optimized for non-Retina Displays. Music MiniPlayer is a completely native app; written with Swift and AppKit.

Requires macOS Monterey or newer and Music.app.3
